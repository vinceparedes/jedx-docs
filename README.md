# jedx-docs

MkDocs site for JEDx documentation, published to GitHub Pages via the workflow
in `.github/workflows/deploy.yml`.

## Hybrid edit model

This repo uses a **hybrid** model. Markdown pages are hand-authored, but the
schema and sample-data sections are regenerated from `../DocumentationPackage`
on demand by `scripts/build.sh`.

### Authored — `build.sh` never overwrites these

Edit freely:

- `mkdocs.yml` — site config, nav, theme
- `docs/index.md`
- `docs/requirements.md`
- `docs/architecture.md`
- All images under `docs/assets/images/`
- Any new pages you add under `docs/`

### Mechanical — regenerated by `build.sh`

Don't hand-edit; changes get wiped on the next build:

- `docs/data-model.md`  ← combined page sourced from `D_Schemas/` (schemas listing) and `E_SampleData/` (Sample Data subsection at the bottom)
- `docs/assets/data-model/`  ← schema files copied from `D_Schemas/`
- `docs/assets/zips/sample-data.zip`  ← packed from `E_SampleData/`

To refresh after dropping new schemas in `D_Schemas/` or updating
`E_SampleData/`:

```sh
./scripts/build.sh
git add docs/ && git commit -m "Refresh schemas/sample-data" && git push
```

## Local preview

```sh
source .venv/bin/activate
mkdocs serve     # http://127.0.0.1:8000/
```

Hot-reloads on file changes. `Ctrl+C` to stop.

## First-time setup on a new machine

```sh
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
brew install pandoc   # only needed if you reintroduce docx conversion
```
